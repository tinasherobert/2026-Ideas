# Security Policy

## Supported versions

We aim to address reported security issues for the current state of the repository (default branch). This is primarily a documentation and ideas repository; there is no deployed application or user data.

## Reporting a vulnerability

If you believe you have found a security vulnerability:

1. **Do not** open a public issue for it.
2. **Email** the maintainers (or open a private security advisory on GitHub if the repo supports it) with:
   * A clear description of the issue
   * Steps to reproduce (if applicable)
   * Impact and suggested fix (optional but helpful)

We will acknowledge your report and respond as soon as we can. We may ask for more detail. If we accept the report, we will work on a fix and coordinate disclosure (e.g. in release notes or a security advisory) as appropriate.

## Scope

* **In scope:** Security issues in any scripts, configs, or tooling that might be run from this repo, and any process or dependency that could affect users who clone or use this repository.
* **Out of scope:** Purely theoretical issues with no practical impact, or issues in third-party services or products that are only referenced in the documentation.

Thank you for helping keep this project safe.
