# Contributing to 2026 Ideas Board

Thanks for your interest in contributing. This repository is a curated list of project ideas and documentation for 2026. Contributions that add or improve ideas, scenarios, or recommendations are welcome.

## How to contribute

### Adding a new idea

1. **Create a document** in the [`docs/`](docs/) folder.
   * Use a clear filename, e.g. `IDEA_NAME_OR_SERVICE.md` or `SERVICE_SCENARIO.md`.
   * Write in Markdown.

2. **Structure the doc** so it’s easy to follow:
   * Short overview and goals
   * Scope and assumptions
   * Worked examples or scenarios if relevant
   * Links to related docs (e.g. other ideas or scenarios)

3. **Update the main README** ([`README.md`](README.md)):
   * Add a new numbered section under **Ideas** with:
     * Idea name and short description
     * Status (e.g. Documented)
     * Link to the new doc
     * Optional: table of sub-documents or scenarios

4. **Update the docs index** ([`docs/README.md`](docs/README.md)):
   * Add an entry and link for the new document (and any sub-docs) in the right place.

### Adding a recommendation

* Put recommendation docs in [`docs/recommendations/`](docs/recommendations/).
* Use a descriptive filename, e.g. `COMPANY_OR_SERVICE_STRATEGIC_RECOMMENDATION.md`.
* Optionally add a link from the main README or docs README if it’s a major recommendation.

### Improving existing docs

* Fix typos, clarify wording, or add sections via direct edits.
* Keep the same style and structure as the rest of the repo (Markdown, clear headings, tables where useful).

## Pull request process

1. **Fork** the repository and create a branch from `master` (or the default branch).
2. **Make your changes** (new docs, README updates, or edits).
3. **Open a pull request** and fill in the PR template.
4. **Describe** what you added or changed and why.
5. A maintainer will review and may suggest changes before merging.

## Code of conduct

This project adheres to the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## Questions

If something is unclear, open an [issue](https://github.com/tinasherobert/2026-Ideas/issues) with your question or suggestion.
